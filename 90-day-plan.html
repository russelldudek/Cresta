First 90 daysIndependent candidate vision by Russell Dudek

Learn the system.
Improve with evidence.

Hypotheses for discovery—not a prescription written without Cresta's internal context.

The first ninety days should earn the right to improve the system: observe how value currently moves, test one useful review pattern, then make proven learning reusable.

Operating principles

Listen firstLearn the current delivery system from the people running it.
Protect momentumAdd clarity where it reduces delay; avoid process theater.
Use live workTest hypotheses against actual deployment decisions.
Leave signalMake each engagement improve the next decision.

Three horizons

Days 1–30

Learn the deployment system.
Observe handoffs, decision rights, product rhythms, customer pressures, and the signals the team already trusts.

Output: shared view of where value stalls

Days 31–60

Prove the Activation Review.
Apply the four-condition model to live work and test whether it creates earlier decisions.

Output: one review pattern worth repeating

Days 61–90

Make learning reusable.
Propose portfolio health signals and a feedback path across delivery, product, success, and GTM.

Output: clearer intervention and expansion decisions

Questions that govern the entry

Customer boundary

  • What do customers mean by ready, adopted, safe, and valuable?
  • Where does decision latency create more risk than technical complexity?
  • Which adoption signals predict business movement?

Internal boundary

  • How do Deployment, FDE, Product, Success, Value, and GTM divide ownership?
  • Which learning is reusable and which must remain customer-specific?
  • Where does Conductor change the deployment bottleneck?

Restraint

No premature redesign.

The plan does not assume Cresta lacks a deployment system. It proposes a disciplined way to learn the existing one, contribute to live delivery, and test improvements with the team's guidance.

02

Days 31–60 · Prove the Activation Review

Use live work, not a workshop toy
Workstream
Actions
Evidence / output
Select
With team guidance, choose one live deployment decision where outcome, readiness, adoption, or evidence is meaningfully uncertain.
A real decision with a named owner and time horizon.
Frame
Run a short Activation Review: define the intended outcome, weakest condition, ownership gap, and evidence required.
One-page review; no duplicate project plan.
Act
Facilitate the next decision, remove or escalate a dependency, and make the customer-facing consequence explicit.
Decision latency, owner clarity, and follow-through observed.
Validate
Ask participants whether the review improved the decision enough to justify repeating or simplifying it.
Keep, change, or stop decision with direct feedback.
Day-60 restraint

A neat artifact is not success. Repeat only if the review creates earlier clarity, cleaner ownership, or better evidence without slowing delivery.

Day-90 restraint

Do not standardize what has only worked once. Preserve exceptions that reveal meaningful customer risk or value.
